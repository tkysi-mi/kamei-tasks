## 📌 このツールでできること

GitHub Projectsは**タスクをカンバンボード（付箋を並べるイメージ）で管理するツール**です。

- タスクを「カード」として登録
- ステータス（バックログ → 進行中 → 完了）を**ドラッグ&ドロップ**で変更
- プロジェクト別にフィルタして、今何に取り組んでいるか一覧で見える

**こんな悩みを解決します：**
- 「あれ、あのタスクどうなったっけ？」 → ボードを見れば一目でわかる
- 「相手の返事待ちのタスクが埋もれてる」 → 「相手ボール」ステータスで可視化
- 「どのプロジェクトのタスクが多いかわからない」 → プロジェクト別フィルタで確認

---

## 🏗️ ボードの構成

### ステータス（4つ）

タスクは以下の4つの状態を移動していきます：

| ステータス | 色 | 意味 | 例 |
|-----------|-----|------|-----|
| **バックログ** | ⬜ グレー | まだ着手していない・いつかやる | 「A社の提案書を作る（来月）」 |
| **進行中** | 🟦 青 | 今やっている | 「B社の見積もり作成中」 |
| **相手ボール** | 🟧 オレンジ | 相手の返事・確認待ち | 「C社にメール送った、返事待ち」 |
| **完了** | 🟩 緑 | 終わった！ | 「D社の契約書提出済み」 |

> 💡 **ポイント**: 「相手ボール」がこのボードの特徴です。待ち状態のタスクを「進行中」に置いておくと、自分のタスクと混ざって見づらくなります。相手待ちのものは積極的に「相手ボール」に移しましょう。

### プロジェクト（5つ）

各タスクには「どのプロジェクトか」を設定できます：

| プロジェクト | 色 |
|-------------|-----|
| **Readable** | 🔵 青 |
| **バクアゲ** | 🔴 赤 |
| **コンサル** | 🟣 紫 |
| **Cona** | 🟡 黄 |
| **その他** | ⬜ グレー |

## 🎯 基本操作ガイド

### 1. ボードを開く

**URL**: https://github.com/users/tkysi-mi/projects/1

ブックマークしておくと便利です。

### 2. タスクのステータスを変更する

**方法A: ドラッグ&ドロップ（おすすめ）**
1. カードをマウスで掴む
2. 移動先のカラムにドロップする
3. 完了！

**方法B: カードを開いて変更**
1. カードをクリックして開く
2. 右サイドバーの「Status」を変更
3. 変更は自動保存されます

### 3. 新しいタスクを追加する

#### 方法A: ボード上で直接追加（シンプルなタスク）

1. 「バックログ」カラムの一番下にある「＋ Add item」をクリック
2. タスク名を入力してEnter
3. カードをクリックして詳細・プロジェクトを設定

#### 方法B: Issue Formから追加（詳細なタスク・他の人からの依頼）

1. https://github.com/tkysi-mi/kamei-tasks/issues/new/choose にアクセス
2. 「📝 タスク依頼」の「Get started」をクリック
3. フォームに入力：
   - **タスク名**: 何をするか（必須）
   - **プロジェクト**: どのプロジェクトか（必須）
   - **詳細・背景**: 追加情報があれば
   - **希望期限**: いつまでか
   - **依頼者名**: 誰からの依頼か（必須）
4. 「Submit new issue」で送信 → **自動的にボードのバックログに追加されます**

> 💡 **他の人からタスクを頼まれた時**: このフォームのURLを共有すれば、相手が自分で依頼を登録できます。

### 4. タスクのプロジェクトを設定する

1. カードをクリックして開く
2. 右サイドバーの「プロジェクト」フィールドで選択
3. これにより、プロジェクト別にフィルタできるようになります

### 5. タスクを絞り込む（フィルタ）

ボード上部の「Filter」を使います：

- **特定プロジェクトだけ見たい**: `プロジェクト is Readable` でフィルタ
- **進行中だけ見たい**: ステータスカラムを見ればOK（カンバンなので自然にフィルタされています）

## 📅 おすすめの使い方（日々のルーティン）

### 朝（5分）
1. ボードを開く
2. 「相手ボール」を確認 → 返事が来てたら「進行中」に戻す
3. 今日やるタスクを「進行中」に移動（2-3件が目安）

### 作業中
- タスクが終わったら → 「完了」にドラッグ
- 相手の返事待ちになったら → 「相手ボール」にドラッグ
- 新しいタスクが発生したら → 「バックログ」に追加

### 週末（10分）
1. 「完了」のタスクを眺めて達成感を味わう 🎉
2. 「バックログ」を見直して、来週やるものを確認
3. もう不要なタスクがあれば削除 or アーカイブ

## ❓ よくある質問

### Q: タスクを間違って完了にしちゃった！
**A**: 大丈夫です。「完了」カラムからドラッグして元のカラムに戻すだけです。

### Q: タスクを削除したい
**A**: カードを開いて、右上の「…」メニューから「Delete from project」を選択。もしIssueとして作った場合は、Issue自体をCloseすればOKです。

### Q: 他の人にもボードを見せたい
**A**: プロジェクトのURLを共有すれば、GitHubアカウントを持っている人なら閲覧できます。編集権限は別途設定が必要です。

### Q: スマホからも使える？
**A**: はい！GitHubのモバイルアプリ、またはブラウザからアクセスできます。ドラッグ&ドロップはブラウザの方がやりやすいです。

### Q: Issue Formから登録したタスクがボードに出てこない
**A**: 自動追加（Auto-add）が設定されています。少し待ってからボードをリロードしてみてください。
